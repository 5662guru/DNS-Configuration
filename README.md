# DNS Configuration and Web Server Access Using Domain Name

## Project Overview
This project demonstrates the configuration of a DNS server in Cisco Packet Tracer, allowing users to access a web server using a domain name instead of an IP address.

## Objective
Configure DNS services and HTTP services so that client PCs can access a website using a domain name.

## Network Topology
- 1 Router
- 1 Switch
- 1 Server
- 2 PCs

## IP Addressing

| Device | IP Address |
|----------|------------|
| Router | 192.168.1.1 |
| Server | 192.168.1.2 |
| PC0 | 192.168.1.10 |
| PC1 | 192.168.1.11 |

## Configuration Steps
1. Create network topology.
2. Configure router IP address.
3. Configure server IP settings.
4. Enable DNS service and create DNS record.
5. Enable HTTP service.
6. Configure client PCs.
7. Verify DNS resolution and web access.

## Verification
- Successful ping using domain name.
- Website accessible through web browser using domain name.
